# 週期角色與映射（整理版）

## 1. 先看角色，不先看絕對 timeframe

三尖理論的本體不是 8H、2H、30m、5m 這些數字本身。
真正重要的是：
- 誰是世界層
- 誰是主戰役層
- 誰是橋接 / 執行層
- 誰是微觀授權層

## 2. 當前 Baseline V1 組合

### 8H：World Credit Layer
- 先看 MACD 綠柱是否收斂，作為止跌背景候選
- 再判 8H 目前在 `3 / +1b / +2` 的哪個時期
- 不直接擔任發單週期

### 2H：Campaign Layer
- 承接 8H 的 `3` 段
- 2H 自己也有 `3 + 1b + 2`
- Baseline V1 的主交易目標只吃 **2H 的 `+2`**

### 30m：Execution Layer
- 用燕返 / 彈切 / 見切做實際進場驗證
- 先看彈切是否突破上 t2
- 再看回測是否守住下 t2
- 主要出場也先用 30m 斷鍊計數

### 5m：Validation Layer
- 只標註 confirmed 斷鍊點
- 只輔助確認最後一個 break point 是否被回穿
- Baseline V1 不讓 5m 變成主交易週期

## 3. 地圖與戰鬥的分工

- **5m / 30m** 是小場戰鬥的細節，負責 gate、點火、局部防守與出場。
- **2H / 8H** 是看清世界的地圖，負責判斷這場仗該不該打、能打多遠、現在屬於主攻還是逆勢偷襲。
- 若沒有 2H / 8H 世界觀，只靠 5m / 30m 無腦操作，在空頭或錯誤世界裡很容易反覆止損。
- 正確順序是：先用 **8H** 看多空世界，再用 **2H** 看 campaign 是否仍合法，最後才讓 **30m / 5m** 決定是否出兵與如何退場。

## 4. 其他常見映射

Mark 實戰曾用過：
- 6H / 1H / 15m
- 1D / 4H / 30m
- 台股：日K 定方向，30m / 60m 找結構，5m 看細節

只要角色分工不變，這些映射都可以成立。

## 5. 角色邊界

- **世界層**：提供主方向、信用與上層授權天花板
- **主戰役層**：主操作與主出場層
- **執行層**：拆解節奏、橋接、handoff
- **微觀層**：點火與最快失敗，不得篡位

## 6. 關於三週期重疊

不要把「三週期完美重疊」寫成死門檻。
較好的理解是：
- 世界層與主戰役層先給出合法背景
- 執行層把時機壓漂亮
- 微觀層負責點火與止損

## 7. 關於延遲補償

- 斷鍊裁決通常比較晚，但比較真
- 多週期驗證的目的，是補回這個延遲
- 不是追求所有級別同時喊同一句話

## 8. 固定提醒

- 小週期不是大結構判官
- 大週期也不是高頻發單層
- 若發現「大一鍊拆成中四鍊」這種怪象，優先懷疑週期挑錯
