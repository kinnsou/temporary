# 週期角色與映射（整理版）

## 1. 先看角色，不先看絕對 timeframe

三尖理論的本體不是 8H、2H、30m、5m 這些數字本身。
真正重要的是：
- 誰是世界層
- 誰是主戰役層
- 誰是橋接 / 執行層
- 誰是微觀授權層

## 2. 04-09 預設組合

### 8H：World Credit Layer
- 提供信用、容錯、母世界框架
- 不直接擔任頻繁發單週期

### 2H：Campaign Layer
- 主操作層
- 判斷合法 `3`
- 審查 `+1b`
- 決定主進場、主出場、重建與續戰

### 30m：Bridge / Execution Layer
- 把 2H 事件拆成可操作節奏
- 判斷現在是 3 鍊、6 鍊，還是只剩短脈衝
- 提供更漂亮的點火與退出節奏

### 5m：Micro Trigger Layer
- 微觀 Double Mikiri 審核
- 最短戰術止損
- 微觀第三鍊交代時快速離場

## 3. 其他常見映射

Mark 實戰曾用過：
- 6H / 1H / 15m
- 1D / 4H / 30m
- 台股：日K 定方向，30m / 60m 找結構，5m 看細節

只要角色分工不變，這些映射都可以成立。

## 4. 角色邊界

- **世界層**：提供主方向、信用與上層授權天花板
- **主戰役層**：主操作與主出場層
- **執行層**：拆解節奏、橋接、handoff
- **微觀層**：點火與最快失敗，不得篡位

## 5. 關於三週期重疊

不要把「三週期完美重疊」寫成死門檻。
較好的理解是：
- 世界層與主戰役層先給出合法背景
- 執行層把時機壓漂亮
- 微觀層負責點火與止損

## 6. 關於延遲補償

- 斷鍊裁決通常比較晚，但比較真
- 多週期驗證的目的，是補回這個延遲
- 不是追求所有級別同時喊同一句話

## 7. 固定提醒

- 小週期不是大結構判官
- 大週期也不是高頻發單層
- 若發現「大一鍊拆成中四鍊」這種怪象，優先懷疑週期挑錯
