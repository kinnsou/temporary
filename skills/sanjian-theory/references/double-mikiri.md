# Double Mikiri（整理版）

> 本檔整理 `trispike_unified_core_strategy_v0_1_2026-04-09.md` 與 `trispike_unified_source_v0_1_2026-04-01.md` 中，真正對 04-09 主幹有用的 Double Mikiri 內容。

## 1. 先定性

Double Mikiri 不是跟 `3 + 1b + 2` 平行的另一套理論。

它更像：

> **原始 `3` 在近距離的事件語言。**

也就是說，它負責把「正在造皇帝位」這件事，拆成更具象、好記、可審計的事件鏈。

## 2. 三個核心事件

### 燕返
- 第一個有效硬見切
- 任務是切出反轉軸
- 它不是最終答案，而是「開始有資格討論 reversal」

### 彈切
- 對燕返的第一次回升測試
- 用來驗證剛才切出的反轉軸，是否只是局部反彈
- 彈切不是授權完成，它只是第一次資格挑戰

### 見切成功
- 回跌守住燕返起點 / 授權低點後，再次見切成功
- 代表最低點授權完成
- 用角色語言說，就是：**皇帝位形成**

## 3. 與 `3 + 1b + 2` 的對應

### 事件層讀法
- 燕返 + 彈切 + 見切成功 = 一組原始三尖點事件鏈
- 先把它記成「在做一個原始 `3`」

### 血統投影讀法
- 燕返 ≈ `3` 的切出
- 彈切 ≈ `+1b` 的第一次測試
- 見切成功 ≈ `+2` 的起始授權

這就是為什麼 Double Mikiri 會和 `3 + 1b + 2`、皇帝位、將軍位自然對上。

## 4. 真正保留的硬核心

不要把舊版的均線姿態硬規則整包搬回來。

目前保留的硬核心是：
1. 有沒有切出第一個有效反轉軸
2. 彈切是否真的形成測試
3. 回跌是否守住燕返起點 / 授權低點
4. 是否完成再次見切，解出新的起漲點

## 5. 被降級成軟證據的東西

以下可以看，但不是最高法源：
- EMA5 一定要上到 EMA89
- 某條均線一定站上某條均線
- MACD 柱體長得像三塊或兩塊

保留一句就夠：

> **EMA / MACD 可幫忙看轉折感與強弱感，但不能直接宣判 Double Mikiri 成立或失敗。**

## 6. 與皇帝 / 將軍的接法

- Double Mikiri 本質上是在**製作皇帝位**
- 皇帝位做好之後，後面才有將軍授任、補做將軍、Ghost 試煉、主權升格這些事情
- 所以它最重要的價值，不是好記而已，而是它把抽象法源翻成了可觀察的事件順序

## 7. 常見誤判

### 誤判 1：把 Double Mikiri 當成單一 timeframe 自己說了算
錯。
- 它先產生事件
- 但這組事件最後被編進哪個世界，要看更上層血統與主權

### 誤判 2：見切失敗就等於整體翻空
錯。
- 失敗的那組事件，可能只是併回更大燕返 / 燕群
- 小級別失敗，常只是更大級別正在重寫授權

### 誤判 3：只靠 EMA 外觀做硬判
錯。
- 容易把 visual pattern 誤當成法源
- 也最容易引入過擬合與偷看未來

## 8. 程式化時先處理的問題

若要寫 Double Mikiri auditor，優先釐清：
- 反轉軸的量化條件
- 彈切力道的分級標準
- 見切失效的精確判定
- 哪些條件是硬條件，哪些只是加分項

目前不要急著把它做成全自動交易器。
先讓程式學會回答：

> **這裡算不算燕返？算不算彈切？見切到底成功了沒？**
