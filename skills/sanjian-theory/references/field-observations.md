# 市場觀察與待驗證假說

> 本檔用來存放 Mark 最新提出、但尚未正式升格為主幹法源的觀察點。
> 原則：可以拿來指引走圖、標記案例、設計審計欄位，但在完成足夠驗證前，不直接覆蓋 `core-rules.md` / `baseline-v1-8h-2h-30m.md` 的硬規則。

## 2026-04-12：多週期同步斷鍊與 MACD 翻色

### 狀態
- 類型：重要市場觀察
- 驗證狀態：待案例驗證
- 目前適用範圍：Baseline V1 的 8H / 2H / 30m
- 5m：先不拿 MACD 當法源
- 15m：目前觀察也偏不靈敏，不宜拿來承擔斷鍊翻色判讀

### 觀察 1：未收盤級別也可能出現「三重斷鍊壓力」

當 30m 準備斷鍊時，若 2H / 8H 雖然尚未收盤，但結構上也同時看起來逼近斷鍊，會出現類似週期重疊的放大效果，可先暫記為：

- **三重斷鍊壓力**
- 常對應深不可測的大跌風險
- 實務上可視為空方時機判讀的重要候選

#### 盤面特徵
- 30m 可能會刻意連續上攻，長時間一直不收綠
- 這種走法會把斷鍊點的 **t2 拉到較遠的下方**
- 因此一旦真正形成斷鍊，深度常會特別大

#### 延伸判讀
- 若 30m 的斷鍊點形成後，止損距離過大，不能只看 30m 自己的事件強度
- 這時要回頭檢查 2H 是否也對應同步逼近或形成斷鍊
- 若 **2H 沒有同步斷鍊**，傾向把它視為 30m 單層事件，代表執行層雖然危險，但未必值得硬扛過大的止損
- 若 **2H 也同步斷鍊或高度逼近斷鍊**，則可把這個 30m 斷鍊升級成 campaign 級別破壞候選，允許提高危險性評分，並作為放空窗口的重要候選

#### 可量化方向
- 在 30m 出現「準備斷鍊」訊號時，同步檢查 2H / 8H 是否也接近各自的斷鍊點
- 若三者在時間與結構位置上高度重疊，可提高危險性評分
- 若 30m 斷鍊對應的 stop distance 過大，可加入 `requires_2h_sync_break` 這類審計條件，作為是否採信該訊號的加權依據
- **注意**：2H / 8H 若尚未收盤，只能當作風險加權與提早預警，不能直接視為 confirmed 斷鍊

### 觀察 2：斷鍊後，MACD 理應翻綠；若不翻反紅，幾乎可視為反攻

在 30m / 2H / 8H 上，Mark 的觀察是：

- 斷鍊之後，最終幾乎都會反映成 MACD 翻綠
- 也就是說，**斷鍊應該要翻綠**
- 若盤面進入「理應翻綠」的區段，MACD 卻反而重新形成紅柱，則幾乎可以視為反攻訊號

#### 目前限制
- 5m 暫時沒辦法穩定使用這條 MACD 判讀
- 15m 目前觀察也常不反應斷鍊後應翻色的情況
- 也就是說，週期太小時，MACD 會變得不夠靈敏，不能把它當硬法源

#### 可量化方向
- 為每次斷鍊建立 `expected_macd_flip_green` 類型的審計欄位
- 若在應翻綠區卻轉為紅柱，可標記為 `counterattack_candidate`
- 後續需統計不同級別上的成功率與延遲時間

### 觀察 3：同步斷鍊的重疊位置，常落在第二鍊 / 核心鍊

Mark 補充觀察：多週期斷鍊點的重疊，常不是落在任意一鍊，而是落在 **第二鍊，也就是核心鍊**。

更精確的延伸是：

- 當**較大週期的核心鍊做完後**，市場可能還會再續攻出一段「第二鍊」
- 等這個**續攻第二鍊發生斷鍊**時，會再觸發一次**核心鍊判定**
- 這次判定的目的，不是立刻宣布翻轉或續勢，而是檢查它**有沒有可能再完成最後一鍊**

也就是說，同步斷鍊在這裡更像是：
- **再核心鍊判定的觸發點**
- 用來審查後續是否仍具備完成最後一鍊的資格
- 不是單靠事件本身就直接下結論

#### 要解的實戰問題
- 理論上市場應該 **三鍊才算完成**
- 但實務上常會在 **第二鍊就先動手腳**，因為一部分持有者已經拿不住
- 因此這套觀察真正要解的是：
  - 這次第二鍊的提前破壞，只是洗盤 / 提前清人
  - 還是已經真的取消了最後一鍊的資格

#### 可量化方向
- 為同步斷鍊事件補一個 `overlap_chain_rank` 或 `is_core_chain_overlap` 類型欄位
- 增加 `post_core_second_leg_break` / `recore_chain_review` 一類審計標記
- 把「是否發生在較大週期核心鍊完成後的續攻第二鍊」納入是否仍有資格完成最後一鍊的分類邏輯

### 觀察 4：成功的 +2 會把火力一路下放，形成多週期遞降接力，最後在 5m 趕工式 +2 後衰竭

Mark 補充的新觀察是，這種「第二鍊 / 核心鍊資格審核」的場景，除了能從同步斷鍊去看，還能從 **多週期遞降的燕返 → 彈切 → 見切 → +2 接力** 去看力道是不是快用完。

#### 典型流程
- 先以 **8H 的上漲段後回檔** 作為大背景
- 在 **8H 下跌的這一段**，常會先走出 **2H 燕返**，也就是 EMA5 穿下 EMA89 到相對低位
- 接著發動 **2H 彈切再下跌**，這一段可用上下 t2 框架去驗證
- 然後進入 **2H 見切 > +2 上攻**
- 若這個 **2H +2** 成功拉高，之後常會再發動 **30m 的燕返 > 彈切 > 再次 t2 驗證 > 見切 > +2 上攻**
- 若 **30m +2** 也完成拉高距離，則可能再往下縮到 **15m 的燕返 > 彈切 > 淺 t2 驗證 > 見切 > +2 上攻**
- 最後再縮到 **5m 的燕返 > 彈切 > 幾乎不驗下 t2 的趕工式 +2**，隨後就容易直接爆跌

#### 詮釋
- 這不是每一級都在重新宣告母世界成立，而是 **火力一路下放到更小週期的接力演示**
- 週期越往下，**t2 驗證常越淺、越趕工**，代表力道品質在下降
- 最後的 **5m +2** 不能輕易當成健康延伸，反而可能是 **終端衝刺 / 衰竭展示**
- 這條觀察**不等於提升 15m / 5m 的法源地位**，它們在這裡比較像是衰竭症狀層，不是主權裁決層

#### 與早上觀察的連動方式
- 早上的觀察是看 **30m / 2H / 8H 的第二斷鍊 / 核心鍊重疊**
- 這條新觀察是看 **2H → 30m → 15m → 5m 的 +2 接力是否一路縮時、一路變淺**
- 若兩者同時出現，也就是：
  - 上層進入第二鍊 / 核心鍊資格審查區
  - 下層又出現一路縮時到 5m 的趕工式 +2
- 那就可把它視為 **力道衰竭 + 同步斷鍊風險共振** 的高危險候選場景

#### 可量化方向
- 為案例補一條 `relay_tf_sequence`，記錄是否出現 `2H -> 30m -> 15m -> 5m`
- 為每一層補 `t2_validation_depth`，比較 t2 驗證是否越來越淺
- 增加 `final_5m_hurry_plus2` 或 `terminal_plus2_candidate` 類型欄位
- 和早上的 `is_core_chain_overlap` / `post_core_second_leg_break` 一起交叉標記

### 觀察 5：第二斷鍊可能不是結案，而是進入上下 t2 測試的前哨

Mark 進一步提出一個很關鍵的相位假說：

- **第二斷鍊**，未必是在宣布結束
- 它更可能是在說，接下來要進入一次 **上下 t2 測試**
- 若這個測試成立，則後面的 **見切** 會成為發動第三大鍊的起點

#### 相位分工的工作定義
- **燕返**：先穩定局勢，切出可重新討論的反轉軸
- **彈切**：進行上下 t2 的測試，確認這次回穩是不是只是局部反彈
- **見切**：若測試通過，成為第三大鍊的發動起點

#### 遞迴特性
- 這個分工不只可能出現在大鍊，也可能在後續拆出的 **三小鍊** 裡重演
- 也就是說，主體若進入第三大鍊，內部小結構仍可能在自己的第二鍊再次進行 t2 測試
- 因此這更像是 **跨週期統一行為**，不是只屬於單一級別的偶發現象

#### 與前面觀察的接法
- 觀察 3 強調：第二鍊 / 核心鍊的斷鍊，常是「最後一鍊資格審查點」
- 這條新假說則更進一步提出：
  - 第二斷鍊之後，市場常不是直接判生死
  - 而是先進入 **t2 上下測試程序**
  - 測試通過後，才由見切去點燃第三大鍊
- 這樣一來，第二斷鍊、t2 測試、見切啟動，就能串成一套更完整的事件鏈

#### 可量化方向
- 增加 `entered_t2_test_after_second_break` 類型欄位
- 為燕返 / 彈切 / 見切補 `phase_role`，明確標記誰在穩勢、誰在測試、誰在點火
- 補 `third_leg_launch_after_mikiri`，統計見切後是否真的進入第三大鍊
- 觀察這套流程是否會在更小週期重演，形成可遞迴辨識的模板

### 後續驗證建議
- 補收 30m 準備斷鍊時，2H / 8H 同步逼近斷鍊的歷史案例
- 區分「已收盤 confirmed 斷鍊」與「未收盤預警斷鍊」兩類訊號
- 統計 MACD 翻綠與反攻訊號的時間差，避免把陰影層誤升格成最高法源
- 補統計同步斷鍊發生在第幾鍊，確認「核心鍊重疊」是否真的顯著高於其他位置
- 補統計 `2H -> 30m -> 15m -> 5m` 的接力案例，確認 t2 驗證是否真的有逐級變淺、最後 5m +2 是否顯著容易接爆跌
- 補統計第二斷鍊後進入 t2 測試的比例，確認燕返 / 彈切 / 見切 是否能穩定對應到穩勢 / 測試 / 點火三種角色
